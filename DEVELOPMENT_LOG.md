# 🛡️ 투두RPG 개발 요약 기록 (DEVELOPMENT_LOG)

## 1일차

### 1번째: 기초 설계 및 도메인 모델링
- **목표**: 클린 아키텍처 기반의 RPG 핵심 로직 수립.
- **핵심 구현**: `User`, `Quest` 모델 생성 및 캡슐화 적용. 게임 상태의 안전한 제어를 위한 커스텀 예외(`골드부족`, `사망`) 설계.
- **성과**: 개발 초기부터 비즈니스 로직과 UI 레이어를 엄격히 분리하여 확장성 확보.

### 2번째: 데이터 영속성 계층 (Repository)
- **목표**: 고성능 데이터 저장 및 조회 시스템 구축.
- **핵심 구현**: JSON 파일 입출력을 위한 `BaseRepository` 개발. 리스트 순회($O(n)$)가 아닌 딕셔너리 키 인덱싱을 통한 $O(1)$ 검색 최적화.
- **성과**: 퀘스트 데이터가 늘어나도 성능 저하 없는 빠른 데이터 처리 보장.

### 3번째: 서비스 레이어 및 비즈니스 로직
- **목표**: 핵심 게임 서비스 연동 및 유효성 검증.
- **핵심 구현**: 보상/레벨업/회복 로직을 담당하는 `RpgService` 및 `ShopService` 구축. 공백 입력 및 잘못된 난이도를 차단하는 `Validator` 구현.
- **성과**: 의존성 주입(DI) 패턴을 사용하여 로직 간 결합도를 낮춤.

### 4번째: CLI 대시보드 완성
- **목표**: 터미널 기반의 시각적인 UI 조립.
- **핵심 구현**: `os.system("clear")`와 특수문자(`■`, `□`)를 활용한 실시간 HP/XP 게이지 바 대시보드 구현.
- **성과**: 터미널 환경에서도 게임의 몰입감을 주는 인터랙티브 UI 달성.

## 2일차

### 5번째: 웹 환경 전환 (Flask 도입)
- **목표**: CLI 핵심 로직을 웹 서버 환경으로 이식.
- **핵심 구현**: `app.py`를 통한 Flask 서버 초기화. 기존 서비스/리포지토리 로직을 100% 재사용하여 웹 API 엔드포인트와 연결.
- **성과**: 코어 로직의 변경 없이 UI 레이어만 교체함으로써 아키텍처의 유연성 입증.

### 6번째: 웹 대시보드 UI (Jinja2)
- **목표**: 템플릿 엔진을 활용한 시각적 대시보드 구축.
- **핵심 구현**: 현대적인 카드 스타일 CSS 디자인 적용. 터미널 게이지를 HTML5 프로그레스 바로 교체하고 기본 퀘스트 관리 기능 연동.
- **성과**: 텍스트 입력 중심에서 마우스 클릭 기반의 직관적인 사용자 경험으로 전환.

### 7번째: RESTful API 및 비동기 UI (AJAX)
- **목표**: 비동기 통신을 통한 대시보드 응답성 극대화.
- **핵심 구현**: 백엔드를 JSON 반환 구조로 개편. 자바스크립트 `fetch` API를 도입하여 페이지 새로고침 없는 실시간 데이터 업데이트 구현.
- **성과**: 퀘스트 완료 및 스탯 변경 시 매끄러운 단일 페이지(SPA) 느낌의 UI 구현.

### 8번째: 환경 트러블슈팅 (WSL & Git)
- **목표**: 개발 환경 오류 해결 및 저장소 최적화.
- **해결**: WSL 환경의 VS Code 확장 프로그램 연동 오류 및 리눅스 전용 `python3` 명령어 인식 문제 해결.
- **해결**: `.gitignore`를 생성하여 1,000개가 넘는 불필요한 가상환경 및 캐시 파일 추적 차단.
- **성과**: 안정적인 개발 환경 확보 및 깔끔한 코드 관리 기반 마련.

### 9번째: UI 고도화 (완료 기록 분리)
- **목표**: 진행 중인 작업에 대한 집중도를 높이기 위한 레이아웃 개선.
- **핵심 구현**: 백엔드 사전 데이터 분류 로직 추가. JS를 이용해 완료된 퀘스트를 '히스토리' 섹션으로 즉시 이동시키는 동적 DOM 조작 구현.
- **성과**: 목록이 길어져도 활성 퀘스트를 한눈에 파악할 수 있도록 가독성 개선.


## 3일차

### 10번째: 미니 상점 시스템 통합
- **목표**: 게임 내 재화(골드)와 실제 보상의 연결.
- **핵심 구현**: `ShopService`를 웹 API와 연동. '유튜브 보기', '소설 읽기' 등 사용자가 정의한 보상 아이템 구매 기능 추가.
- **성과**: '퀘스트 수행 -> 보상 획득 -> 보상 소비'로 이어지는 핵심 게임 루프 완성.

### 11번째: 통합 예외 처리 및 사망 페널티
- **목표**: 백엔드 예외와 프론트엔드 알림의 동기화.
- **핵심 구현**: 공통 에러 핸들러(`handleResponse`) 도입. HP 0 도달 시 "사망 및 스탯 초기화" 로직(Lv 1로 리셋) 및 자동 페이지 새로고침 구현.
- **성과**: 명확한 유저 피드백을 통해 게임의 긴장감과 규칙 엄격성 강화.

### 12번째: 반응형 웹 디자인 (Mobile First)
- **목표**: 모바일 기기에서의 쾌적한 플레이 환경 보장.
- **핵심 구현**: CSS 미디어 쿼리 적용. 화면 폭에 따라 그리드 레이아웃을 전환(2열 -> 1열)하고 테이블의 가로 스크롤 대응.
- **성과**: 언제 어디서든 스마트폰으로 투두를 관리할 수 있는 모바일 최적화 달성.

### 13번째: 실시간 정렬 엔진
- **목표**: 방대한 퀘스트 목록의 효율적인 관리.
- **핵심 구현**: 자바스크립트 기반 정렬 기능 구축. 난이도순(상/하), 등록순 정렬을 위한 HTML5 `data-*` 속성 메타데이터 활용.
- **성과**: 서버 요청 없이 브라우저 내에서 즉각적으로 우선순위 작업을 찾을 수 있도록 개선.

### 14번째: 대시보드 레이아웃 전면 개편 (2분할 카드)
- **목표**: 정보 가독성 극대화 및 가로 스크롤 완전 제거.
- **핵심 구현**: 모든 테이블 태그를 제거하고 Flexbox 기반 카드 스타일로 교체. 데스크톱 기준 2분할(스탯/퀘스트 vs 상점/기록) 그리드 적용.
- **성과**: 모바일 가독성 문제 해결 및 전문적인 게임 대시보드 스타일의 비주얼 완성.
